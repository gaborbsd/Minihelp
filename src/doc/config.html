Configuring Minihelp

Minihelp displays single HTML documents in an organized way. To describe the documents, the relations between them and the index keywords, you need edit an XML file. In this section, we will review the structure of this XML file. There is no editor provided, the configuration file must be written by hand, but the structure is very simple. The root element is called <minihelp>. Inside this, the following elements can be used:

  • title

    Content: the main title of the documentation set.

  • homeID

    Content: the identifier of the document that must be shown after startup.

  • map

    No content.

    Attribute target: an identifier that will be used to refer to a specific resource.

    Attribute url: the URL of the document to associate to the specified identifier.

  • tocitem

    Content: optionally, child TOC items may be specified under a particular item.

    Attribute target: the identifier of the document to open when the particular TOC item is selected.

    Attribute text: the title of the particular TOC item.

  • indexitem

    Content: optionally, child index items and index entries.
